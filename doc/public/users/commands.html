Pak Commands

Installing Paks

To install named paks:

pak install PAK...

A PAK can be the name of a package published in the Pak catalog a GitHub repository endpoint or a path to a local pak directory. 

pak install jquery
pak install https://github.com/embedthis/jquery.git
pak install embedthis/jquery

A PAK name may have a "#version" suffix that nominates a specific version to install. All required dependent paks will be automatically installed.

To install all the paks required by an application as specified in package.json:

pak install

Removing Paks

To uninstall a pak from your local directory. The pak will remain in the local Pak cache.

pak uninstall PAK...

List Paks

To list installed paks in your local directory:

pak list

The list will display the pak name and installed version. Paks may designated as optional if the package.json describes them in the 'optionalDependencies' list. Paks may be shown as 'foreign' if they are installed but not present in the package.json.

Pak Dependencies

To display dependencies for installed paks in your local directory

pak depend

This will display the installed paks and their dependencies with versions.

Cached Paks

When pak installs a package, it keeps a copy of the pak in the Pak cache which is typically at "~/.paks". The cache stores all versions downloaded of a Pak.

pak cached

Pre-Caching

To download and cache a pak but not install in the local directory

pak cache Name

Pruning Paks

To prune old versions from the cache

pak prune
