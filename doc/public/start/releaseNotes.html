Release Notes

Welcome to the Embedthis Pak™ installation release notes. This document provides release-specific information for Pak. This package contains a pre-built stand-alone (binary) web server and the development libraries and headers needed to embed Pak in a device or application.

License

This software is copyrighted and distributed under license. It is available under a GPLv2 open source license and an optional commercial license. Please read LICENSE.md for details.

Pak includes support for several external packages such as MatrixSSL, OpenSSL, and PHP. These packages have their own licenses. Make sure you check the relevant package license details to ensure you are complying with their licenses.

System Requirements

Binary releases are supplied for the following operating systems.

  • Windows (x86)
  • Mac OS X (Mountain Lion-x64)

The following operating systems may be supported by recompiling Pak from source.

  • FreeBSD 10.X
  • Linux 2.6 and later
  • Mac OS X 10.8 and later
  • VxWorks 6.X
  • Windows 7 and later

The following CPU architectures are supported: arm, mips, ppc, xscale, x86, x64, sparc.

Run-time System Requirements

To deploy Pak in an embedded environment, you will need at least the following:

  • 1-2 MB Disk/ROM space — depending on the application size
  • 1-4 MB RAM

Development Environment Hardware Platform Support

If you wish to build Pak from source code, you will need to use a "C" compiler and associated development tools. Several development environments are supported. You may choose any of the following to compile and build samples and source code.

  • Linux GNU tools
  • Windows Visual Studio 10 (express editions are fine)
  • Mac Xcode
  • WindRiver Tornado Development Tools.

Helpful Hints

Debug Trace and Logging

Pak has logging and request trace facilities that can help when debugging configuration issues or developing your application code that uses Pak. You can vary the level of logging via the --log logfile command line switch for the pak commands. The log command line switch has the format:

pak --log logFile:logLevel

Use the -v switch to trace to the standard output at level 2.

pak -v

The log level can be from 0 to 5, with 5 being the most verbose level.

Similarly you can emit HTTP request trace using the --trace switch.

pak --trace traceFile:traceLevel

The -v switch will emit request trace to the standard output at level 2 combined with the error log.
